# Roadmap

## Phase 1: Foundation

- Establish repository structure.
- Define naming conventions for Pine Script modules.
- Document the intended market structure concepts.
- Add non-trading scaffolds for core components.

## Phase 2: Core Market Structure

- Define swing detection requirements.
- Define trend state requirements.
- Define support and resistance concepts.
- Add visual diagnostics for development.

## Phase 3: Strategy Development

- Introduce strategy prototypes in `pine/strategy`.
- Keep strategy behavior separate from reusable core logic.
- Document assumptions, limitations, and validation criteria.

## Phase 4: Release Preparation

- Prepare cleaned scripts in `pine/release`.
- Add version notes and release checklists.
- Capture examples and screenshots in `docs` and `images`.

## Current Focus

Initial project setup only. No trading logic is implemented yet.
